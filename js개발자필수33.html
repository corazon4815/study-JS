IIFE(Immediately Invoked Function Exprssion(즉시호출되는함수식))와 모듈> TCP는 서버와 클라이언트간에 계속 상호작용을 함 UDP는 데이터받으면서 유실이 생기지만 빠르게 받음 인터넷 - 서버와 클라이언트, 클라이언트와 클라이언트간의 응답과정 무엇을 이용해 통신할까? 이 매체들은 선으로 연결 돼있음 -> world wide web 거미줄 -> 이구조를 네트워크라고함 인터넷을 크게봐서 네트워크라고함 TCP/IP 프로토콜 - 인터넷을 할때 사용하는 규칙 OSI7 Layer Divide and 두개의 시스템이 통신하기 위한 7가지 절차 application Layer - 전달하고자하는 내용을 컴퓨터에 씀 presentation Layer - 표현하는 계층 데이터를 Encoding, 암호화, 압축 사진을 보냈는데 동영상이라고 받으면 안됨 영어를 보냈는데 중국어를 보낸 것처럼 받으면 안됨 보내는 데이터를 표준형식에 맞춰 보내는것 아라비아 숫자같이 누구나 알수있는 표현으로 보내는것 session Layer - 전화처럼 쌍방으로 할지 아니면 받기만할지 생성, 유지, 종료 관리를 담당 transport Layer - 누가보냈는지 누가받는지 꼬리표를 달아줌 (port번호) 온전한 데이터를 받을지 책임지는것 TCP(등기우편) (반대 ->UDP(우편)) network Layer - 가야하는 경로를 알아야하는것 경로 설정과 도착지 (ip(인터넷프로토콜) (ex:192.168.0.1)) data link Layer - 아파트의 몇동 몇호 (MAC(Media Access Control) 주소) physical Layer - 디지털신호를 전기신호로 전송하는것 내려오면서 각각의 데이터를 합치는 과정 Encapsulation 수신측에서 편지를 열어보는 과정 Decapsulation
